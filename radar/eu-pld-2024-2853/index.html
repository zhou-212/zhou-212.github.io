欧盟 PLD 产品责任指令 过渡期:2026-12-09 各成员国转化截止,软件/AI 卖家责任全面升级

2024-12-09 生效的新版产品责任指令, 取代 1985 年的旧指令. 首次将软件、AI、数据纳入产品范畴, 卖家需承担举证倒置、心理健康影响、网络安全漏洞等新型责任.

一句话结论

如果你在 Amazon EU 卖实体产品 + 软件 + 联网硬件 + AI 系统,2026-12-09 起 EU 2024/2853 (新版 Product Liability Directive, PLD) 在各成员国国内法转化后正式执行 —— 首次把软件、AI、原始数据、数字化制造文件纳入「产品」定义,举证责任倒置(卖家要自证产品无缺陷),网络安全漏洞、心理健康影响也算「缺陷」,集团诉讼可代位起诉。这是一部 Directive(指令),不是 Regulation —— 各成员国有 2 年时间把它变成国内法,转化版本可能比欧盟原文更严。

适用范围

  • 法规级别:欧盟 Directive(指令) —— 不是 Regulation,需要各成员国在 2026-12-09 前转化为国内法;转化版本因国而异(德国、法国、荷兰可能加严)
  • 覆盖范围:几乎所有投放欧盟市场的产品 + 软件 + AI 系统 + 数字制造文件;取代 1985 年 Council Directive 85/374/EEC
  • 典型适用:
    • 实体产品:消费电子、家电、玩具、纺织、化妆品、机械、车辆
    • 软件(独立软件 + 嵌入式固件 + SaaS 中可分离组件)
    • AI 系统(模型权重、训练数据、推理输出;与 AI Act 协同)
    • 数字制造文件(digital manufacturing files, 3D 打印的 CAD 文件)
    • 原始数据 / 传感器数据(影响产品安全决策的输入数据)
  • 卖家类型:制造商、进口商、授权代表、电商平台、FBA 卖家、OEM/ODM、AI 组件开发者
  • 不适用(典型):核损害、内乱、武装冲突场景;未在欧盟市场投放的纯出口产品

关键时间线

  • 1985-07-25:Council Directive 85/374/EEC 通过(老 PLD)
  • 2024-10-09:European Parliament 投票通过新版 PLD
  • 2024-11-18:Directive (EU) 2024/2853 在 Official Journal 颁布
  • 2024-12-09:生效(entry into force)
  • 2026-12-09:各成员国国内法转化截止(这是卖家真正的 deadline;转化版本同日适用)
  • 2026-12-09 起:消费者 / 集团诉讼可援引新版 PLD 起诉(转化法)
  • 后续 5 年内:European Commission 评估是否进一步覆盖非物质 AI 决策(无具体日期)

核心变化 1 —— 「产品」定义扩展

老 PLD 的「产品」= 动产(可移动的物品);新版扩展为:

  • 软件 = 产品 —— 无论是否嵌入硬件、独立销售、还是 SaaS 中可分离组件
  • AI 系统 = 产品 —— 涵盖模型权重、训练数据集(影响输出安全)、推理接口
  • 数字制造文件 = 产品 —— 3D 打印蓝图、CAD 文件、BIM 模型
  • 原始数据 = 产品 —— 影响产品安全功能的传感器输入、遥测、用户生成数据

关键判断:独立 App、SaaS、嵌入式固件、固件更新包、AI 模型权重,均落入「产品」范畴。这意味着你的每一次软件更新 / 模型重训,都可能产生新的「产品」并触发新的「缺陷」评估窗口。

核心变化 2 —— 举证责任倒置

老 PLD 实行谁主张谁举证;新版引入举证倒置 —— 消费者只要初步举证,卖家需自证清白:

  • 明显缺陷:产品明显不符合同类产品应具备的安全水平,且损害发生
  • 不遵守产品安全法规:产品未满足强制性产品安全要求(CE / GPSR / CRA / 行业法规)
  • 卖家未配合披露技术文档:消费者或法院要求披露时,卖家无理拒绝或延误
  • 损害发生于产品本应保障安全的合理使用场景

后果:卖家从「被动应诉」变成「主动举证」。技术文档、DoC、SBOM、事故记录、补丁日志全部要保存 10 年,缺一项就可能败诉。

核心变化 3 —— 缺陷类别与损害范围

缺陷类别扩展(新版明确把以下列为「缺陷」):

  • 网络安全漏洞 —— 已知未修补的 CVE、未授权访问风险
  • 影响心理健康 —— AI 陪伴机器人、社交 App、青少年保护机制失败的损害
  • 隐私侵犯 —— 与 GDPR 协同,数据滥用导致可识别的非物质损害
  • 歧视性 AI 决策 —— 信用、保险、招聘场景的偏见损害
  • 产品 + 服务组合缺陷 —— 与服务捆绑销售的硬件(订阅制设备)

  • 非物质损害(精神损害、隐私侵犯、心理伤害)可独立主张
  • 集团诉讼 / 消费者协会代位起诉 —— 5 个成员国以上消费者受影响即可
  • 时效:损害显现后 3 年内起诉,产品投放后 10 年内为绝对时效
  • 跨境判决互认:欧盟成员国法院判决在其他国家自动承认 + 执行

核心变化 4 —— 平台与 FBA 责任

  • 电商平台(Amazon EU、TEMU、AliExpress、Shopee EU)**可被视为「卖家」**在满足以下条件时:以自己名义提供产品、对产品安全有决定性影响、促成跨境销售
  • FBA 卖家的责任链:Amazon EU 作为仓储服务方不直接承担责任,但亚马逊平台合规要求(Responsible Person、安全信息上传)与 PLD 卖家责任协同
  • Marketplace 平台需提供:卖家信息透明、缺陷产品下架、不安全产品追溯

合规清单(自检 6 项)

  • 产品清单已按「软件 + AI + 数据」维度重梳:独立软件 / 嵌入式固件 / SaaS 组件 / AI 模型 / 数字制造文件均纳入产品责任评估
  • 举证倒置四情形已内部培训:明显缺陷 / 不遵守安全法规 / 拒绝披露文档 / 损害发生在合理使用场景 —— 售后 / 法务 / 客服团队均能识别
  • 技术文档保存 10 年:DoC、CE 报告、SBOM、事故记录、补丁日志、客服投诉分析,支持 10 年内任何 PLD 诉讼的「自证清白」举证
  • 网络安全漏洞响应机制已建立:与 CRA (EU) 2024/2847 协同,24 小时内 ENISA 通报;产品安全风险评估纳入 SDLC
  • 多语言说明书已覆盖产品销售国语言(DE / FR / IT / ES / PL / NL 等),使用条款 + 风险提示完整
  • 心理健康 / 隐私 / 歧视性 AI 损害评估已做(适用产品):AI 陪伴、青少年产品、推荐算法、社交功能、招聘 / 信用 AI

你现在要做什么(按你的角色)

品牌卖家 / OEM:

  1. 盘点产品清单:含软件 + 固件 + AI 组件,逐 SKU 登记;与 CRA / GPSR / AI Act 清单合并维护
  2. 内部培训:产品 / 法务 / 客服 / 售后 / 营销团队理解「举证倒置」四情形;技术文档保存 10 年的硬性要求写入数据保留政策
  3. 网络安全响应:与 CRA 协同,24 小时内向 ENISA 通报已利用漏洞;SDLC 纳入产品安全风险评估;SBOM 维护
  4. 多语言说明书 + 风险提示:销售国语言全覆盖;含产品预期使用 + 不合理使用边界
  5. AI 损害评估:AI 陪伴、青少年产品、推荐算法、招聘 / 信用 / 保险 AI 做独立产品安全影响评估
  6. 欧代 / Responsible Person 合同条款更新:明确举证配合义务 + 技术文档披露范围 + 召回配合

贸易商 / FBA 卖家:

  1. OEM 协议加严:缺陷连带责任、技术文档保存义务、举证配合条款、召回配合义务写进采购合同
  2. 要求 OEM 提供 DoC + CE 报告 + SBOM + 事故记录;保存 10 年作为「自证清白」证据链
  3. Listing 信息同步:产品责任相关声明(CE 标志、Responsible Person、制造商信息)与 DoC 一致
  4. 客服 / 售后流程:产品安全投诉 / 缺陷反馈走专门通道;24 小时内升级到合规团队

亚马逊相关动作

  • 2025-08 起:Amazon EU 已开始对部分类目(电子、玩具、家居)要求更新产品安全信息,包括Responsible Person + 多语言说明书 + 网络安全声明;此动作先于 PLD 转化落地
  • DoC + CE + 风险评估上传到 Seller Central 合规门户:与 GPSR / CRA / 电池法规 共享同一门户
  • Listing 与 DoC 一致性审查:亚马逊批量合规团队抽查 ASIN,未通过发警告邮件,二次未通过限流
  • AI 增强 Listing 风控:AI 生成的产品声称 + 误导性图片 + 虚假风险提示将受到 Amazon AI 内容审核加强
  • FBA 仓储责任澄清:Amazon EU 作为仓储服务方不直接承担 PLD 卖家责任,但亚马逊对卖家合规要求(产品安全上传、Responsible Person 验证)会持续加严

不处理的潜在影响

  • 高概率:2026-12-09 后欧盟各成员国的产品责任诉讼门槛大幅降低,集团诉讼 + 消费者协会代位起诉将成常态
  • 高概率:跨境电商(纯出口到欧盟的中国卖家)首当其冲 —— 在欧盟无资产、无分支,转化法下成员国法院判决可能在全欧盟互认 + 执行
  • 可能:消费者在欧盟法院起诉后,欧盟成员国互认机制让判决在另一成员国自动生效,海外资产可被申请执行
  • 可能:心理咨询、健康监测、AI 陪伴等新型损害的人身伤害赔偿接近上限 €1.2M / €1.4M
  • 明确:本卡片不构成法律意见;PLD 是 Directive,各成员国转化版本可能加严;具体类目适用、AI 责任边界、集团诉讼程序需查销售国转化法 + 当地律师

常见问题

Q1:我是纯软件(SaaS / App)卖家,PLD 适用我吗? A:适用。新版 PLD 明确把软件 + AI 系统纳入「产品」定义,无论是否嵌入硬件。独立 SaaS(纯云服务,无硬件载体)是否完全适用仍看销售国转化法与服务条款,但嵌入式固件 / 硬件配套 App / 模型权重 几乎肯定在范围内。

Q2:举证倒置后,卖家要保留哪些证据? A:建议保留10 年:DoC + CE 报告 + 测试报告 + SBOM + 漏洞修复记录、产品风险评估报告(含 AI 偏差、心理健康、隐私影响)、客服投诉分析 + 事故记录 + 召回历史、OEM 合同 + 供应链合规协议、多语言说明书 + 用户协议 + 风险提示。缺一项,在转化法生效后被起诉时可能直接败诉。

Q3:心理健康影响也算「缺陷」?哪些产品要特别关注? A:是的。AI 陪伴机器人、青少年社交 App、推荐算法、心理咨询类 AI、内容审核失败的平台 都要特别关注;儿童 / 青少年产品是欧盟 2024-2026 年的重点监管对象。建议做法:心理健康影响评估纳入产品安全风险评估;青少年保护机制单独成段写入说明书;AI 偏差测试周期性做。

Q4:固件更新后产品变化,算「缺陷」吗? A:灰色地带,关键看「合理预期」。新版 PLD 引入**「合理预期安全」标准 —— 消费者对产品的安全预期会随技术演进而变。OTA 固件更新如果未配套安全公告 + 未提供回滚选项 + 引入新功能未告知风险**,可能被视为**「更新引入的缺陷」。建议:重大 OTA 更新配发安全公告 + 风险提示 + 回滚路径**。

Q5:集团诉讼 / 消费者协会代位起诉,真的能告跨境卖家吗? A:能。5 个成员国以上消费者受影响即可触发;欧盟成员国法院判决在其他国家自动承认 + 执行;跨境电商 + 纯出口卖家在欧盟无资产也可被起诉,只是执行时通过国际私法 / 海牙公约。Be careful:中国卖家在欧盟消费者协会发起的跨境集团诉讼中有败诉 + 在欧盟无资产执行风险。
