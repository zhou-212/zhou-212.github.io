如何行动(5 步)

  1. 1

    看你的产品是否在新 PLD 范围内

    PLD 覆盖所有"产品" = 物理商品 + 软件 + AI 系统 + 数字内容 + 集成数据。豁免:纯服务(咨询 / 医疗)、开源软件(非商业用途)、军事 / 国家安全用途。

  2. 2

    准备技术文档(可被法官 / 律师调取)

    PLD 要求"举证倒置" —— 出问题后,卖家要自证产品无缺陷。技术文档包括:设计文档、测试报告、风险评估、版本历史、用户手册。文档保留 10 年,法院 / 律师 / 保险公司可调取。

  3. 3

    升级产品责任险(覆盖软件 / AI 损失)

    传统产品责任险只覆盖"物理伤害 + 财产损失";新 PLD 涵盖"软件故障导致的经济损失 + AI 决策错误 + 数据泄露"。找保险公司明确:是否覆盖软件 / AI / 数据相关索赔。

  4. 4

    建立客户沟通机制(可联系 + 投诉渠道)

    产品文档里要写明:技术负责人姓名 + 联系方式 + 投诉处理流程。出现事故时,买家 / 律师要能直接找到人,不能"找不到责任人"。

  5. 5

    关注 2026-12-09 各国转化后的本地法律变化

    PLD 是"指令",各国自己转化立法。德、法、意、西等国的本地法律可能在细节上有差异(比如损害赔偿上限、举证细节、时效)。建议各国分别看本地法律。

常见问题

Q 我卖纯软件(比如 App / SaaS),也受 PLD 影响吗?
A

受。新版 PLD 明确把软件纳入"产品"范畴,无论独立销售还是嵌入硬件。SaaS 服务的责任边界要看合同条款(免责声明不能完全排除 PLD 强制的责任)。

Q "举证倒置"是什么意思?对卖家有什么影响?
A

旧 PLD:买家要证明产品有缺陷;新 PLD:卖家要自证产品无缺陷(或者买家证明"明显缺陷"即可)。这意味着卖家要保留完整的设计 / 测试 / 风险评估文档,出事后举证责任在你。

Q 我的产品是开源软件,豁免吗?
A

部分豁免。PLD 对"非商业开源"豁免,但"商业开源"或"商业产品 + 开源组件"仍受约束。如果你的产品是商业产品但用了开源组件,你要为整体产品负责,不能把责任推给开源社区。

Q 心理健康影响也成了产品责任?这是什么意思?
A

新 PLD 把"心理健康伤害"纳入损害赔偿范围。如果你的产品(尤其 AI 陪伴、社交产品、儿童玩具)被证明导致用户心理问题(焦虑、抑郁、成瘾),可能面临索赔。建议产品设计时做心理健康影响评估。
